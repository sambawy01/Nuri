## Why This Feature Matters

IXL's best analytics are locked behind school licenses. Parents using the consumer version get limited insights — basic accuracy scores and time spent. That is a moat we can walk right through.

Nuri gives EVERY parent a full analytics dashboard from Day 1. No school license. No premium tier. Every parent sees exactly how their child is learning, what they are struggling with, and what to do about it — explained in plain language by an AI that actually understands their kid.

This is not a data dump. It is an AI-interpreted, actionable, emotionally intelligent view of their child's learning journey. And it includes a viral sharing mechanism that turns proud parents into our best marketing channel.

## Feature 36: Parent Account System 👤

Parent accounts are fully separate from child profiles. The parent never needs to touch the child's device.

### Account Creation

**Flow:**
1. Parent downloads the app (or opens web dashboard)
2. Taps "I'm a Parent"
3. Creates account with email + password
4. Verifies email
5. Optional: enables two-factor authentication (TOTP or SMS)

### Linking to Child Profile(s)

**PIN System:**
1. On the child's device, in Settings → "Link to Parent" → generates a 6-digit PIN
2. PIN is valid for 10 minutes
3. Parent enters PIN in their dashboard → child profile is linked
4. Parent can link multiple children (siblings) to one account
5. Two parents can link to the same child (mum + dad both get insights)

**Important:** The child does NOT need to interact with the parent dashboard at any point. They see a brief "Linked to parent" indicator in settings and nothing else.

### Multi-Parent Support

- Two parent accounts can link to the same child profile
- Each parent has independent notification preferences
- Either parent can set goals (both are visible to both parents)
- Useful for separated/divorced families — both parents stay informed

### Security

- Parent account is password-protected with optional 2FA
- Session tokens expire after 30 days of inactivity
- Child cannot access parent dashboard (separate auth flow)
- Parent cannot modify child's learning data — read-only + goal setting only
- PIN linking prevents unauthorized access to a child's data

## Feature 37: Daily Summary Push Notification 📬

Enhanced version of the original Parent Highlights (Feature 15). Richer data, multiple delivery channels, AI-written insights.

### Nuri's Daily Note

Sent every evening at a configurable time (default: 8:00 PM). Parent chooses delivery channel:

- **Push notification** (app)
- **WhatsApp message** (via WhatsApp Business API)
- **Email** (formatted HTML)
- **SMS** (condensed version)
- Multiple channels simultaneously supported

### Content

```
🦉 Nuri's Note — Tuesday, March 31, 2026

📊 Today's Session:
Youssef studied for 24 minutes today (weekly avg: 18 min ↑)

📚 Subjects Covered:
  • Maths — Fractions: 9/10 correct (90%) ✅
  • Science — Forces & Magnets: Learn Mode, 15 min
  • Arabic — Reading Comprehension: 6/10 correct (60%) ⚠️

🔴 Struggled With:
  Equivalent fractions with unlike denominators — got 2 out of 4 wrong.
  Nuri detected a pattern: he's multiplying only the numerator
  when finding equivalents, not the denominator too.

🌟 Excelled At:
  Times tables — 100% accuracy across all questions today.
  The Water Cycle — explained it back to Nuri perfectly.

😊 Session Mood: Confident
  He answered quickly and used the "Knew it" confidence button
  on 7 out of 10 maths questions. Slight hesitation in Arabic
  (longer answer times, used "Unsure" on 4 questions).

🔥 Streak: 14 days!

💡 Actionable Tip:
  "Try counting coins at the supermarket to practice
  equivalent values — 50p = two 20p + one 10p. Same idea
  as equivalent fractions!"

👨‍👩‍👧‍👦 Co-Learning Moment (5 min):
  He studied magnets today. Grab a fridge magnet and walk
  around the house — what sticks? What doesn't? Ask him WHY.
```

### Notification Rules

- Only sent on days the child actually studied
- If child didn't study: "Youssef didn't open Nuri today. No worries — tomorrow's a new day! 🦉"
- If child studied but only for 2 minutes: shorter summary, gentle encouragement
- Content adapts based on what data is available (no empty sections)

### AI Generation

The daily note is generated by AI (Claude) using structured session data. The prompt includes:

- Today's session data (time, subjects, accuracy, errors)
- Error pattern analysis from Feature 21
- Confidence meter data from Feature 19
- Learning style profile from Feature 17
- The parent's preferred language (English or Arabic)
- Previous notes (to avoid repetitive tips)

## Feature 43: Goal Setting 🎯

Parents can set learning goals for their child. Nuri integrates these goals naturally into the learning experience.

### Goal Types

**1. Daily Study Time Goal**
```
Set a daily study goal:
  [ 10 min ]  [ 15 min ]  [ 20 min ]  [ 30 min ]  [ Custom ]

Currently: 20 minutes/day
Child's actual average: 19 min/day (close! ✅)
```

**2. Weekly Subject Targets**
```
Set weekly subject targets:
  Maths:     [ 3 sessions/week ]  ✅ On track (2 done)
  Science:   [ 2 sessions/week ]  ✅ Completed
  English:   [ 2 sessions/week ]  ⚠️ 0 done
  Arabic:    [ 3 sessions/week ]  ⚠️ 1 done
  History:   [ 1 session/week  ]  ❌ Not started
  Religion:  [ 1 session/week  ]  ✅ Completed
```

**3. Test Prep Countdown**

Linked to Feature 12 (Pre-Test Predictor):
```
📝 Upcoming Tests:
  Science — Forces: April 3 (3 days away)
    Study plan: Day 2 of 4 ✅
    [ View study plan ]

  Arabic — Reading: April 10 (10 days away)
    Study plan: Not started
    [ Create study plan ]
```

**4. Topic Focus Goal**
```
Set a focus topic:
  "Practice equivalent fractions 3 times this week"
  Progress: 1/3 sessions completed
```

### How Nuri Communicates Goals to the Child

Goals are mentioned **casually** — never as parental pressure:

| Goal Type | Nuri Says |
|-----------|-----------|
| Daily time | "You've been studying for 15 minutes — just 5 more and you'll hit today's goal! You're so close! 🌟" |
| Subject target | "You've done 2 Arabic sessions this week! One more and you've hit the target. Want to do a quick one now?" |
| Test countdown | "Your Science test is in 3 days! Today's plan: review magnets. Ready?" |
| Topic focus | "I've got some fraction challenges ready for you — want to give them a go?" |

**Critical rule:** Nuri NEVER says "Your parent wants you to..." or "Your dad set this goal." It is always framed as Nuri's suggestion or the child's own target. This preserves intrinsic motivation.

### Goal Achievement

When a goal is met:
- Child sees: "+20 XP Goal Bonus! 🎯" and a brief celebration from Nuri
- Parent sees: "Goal achieved! ✅" notification
- If ALL weekly goals are met: "Perfect Week" badge for the child

When a goal is missed:
- No punishment. No negative messaging to the child.
- Parent sees: "Arabic target not met this week (1 of 3 sessions). Consider adjusting the goal or talking to Youssef about Arabic."
- Nuri subtly increases nudges for that subject next week

## Feature 44: Emotional & Wellbeing Indicators 💛

What IXL does not provide. Session mood estimation based on behavioral signals — not self-reporting (kids are unreliable self-reporters at this age).

### Mood Estimation Signals

| Signal | What It Means | Data Source |
|--------|--------------|-------------|
| Answer speed | Fast + accurate = confident. Fast + wrong = guessing. Slow + right = thinking hard. | Quiz response times |
| Confidence ratings | "Guessed" vs "Knew it" pattern | Confidence Meter (Feature 19) |
| Session length | Ended early = frustrated or bored? Went long = engaged? | Session timestamps |
| Struggle patterns | 3+ wrong in a row on similar questions | Error tracking |
| Difficulty changes | Dropping difficulty = struggling. Raising difficulty = confident. | Difficulty Dial (Feature 11) |
| Hint usage | Heavy hint use = struggling but trying. No hints + wrong = overconfident or guessing. | Homework Helper data |
| "I don't know" rate | High skip rate = disengaged or lost | Quiz skip tracking |

### Session Mood Categories

| Mood | Indicators | Icon |
|------|-----------|------|
| Confident | Fast answers, high accuracy, high confidence ratings, long session | 😊 |
| Focused | Moderate speed, good accuracy, steady pace, few breaks | 🎯 |
| Struggling but trying | Slow answers, using hints, low accuracy but completing questions | 💪 |
| Frustrated | Multiple wrong in a row, session ended early, speed increasing (rushing) | 😤 |
| Disengaged | Very fast answers, low accuracy, many skips, very short session | 😶 |
| Playful | Using voice a lot, exploring different subjects, trying Challenge Me difficulty | 🎮 |

## Feature 45: Parent-Child Privacy Model 🔒

Critical design consideration that affects every aspect of the parent dashboard.

### Core Principles

1. **The child NEVER sees the parent dashboard.** Different app interface, different auth, different device.

2. **The child doesn't know exactly what data parents see.** They know their parent gets "updates" but not the specifics.

3. **Nuri treats the child as the learner, the parent as the supporter.** Nuri's relationship with the child is direct — not mediated through the parent.

4. **Goals are framed as Nuri's suggestions, never parental demands.** "Want to try Arabic today?" not "Your mum wants you to do Arabic."

5. **Negative data is never weaponized.** The dashboard shows struggles to help parents support — not to punish. The UI reinforces this with messaging like "How to help" rather than "What went wrong."

### What the Child Sees

- A small "Linked to parent" text in Settings (not prominent)
- Goal celebrations when they hit targets ("+20 XP Goal Bonus!")
- Nuri naturally steering them toward focus areas
- Nothing else. No dashboard. No reports. No accuracy percentages shown as parental oversight.

### What the Child Does NOT See

- Their mood/frustration analysis
- The comparison between subjects
- Parent's goal targets (only Nuri's casual mentions)
- Error pattern analysis
- Weekly reports
- Confidence trend analysis
- The "Send a nudge" mechanism

### Why This Matters

Research consistently shows that children who feel surveilled lose intrinsic motivation. If a child knows their parent is watching every quiz score, they will:
- Avoid hard subjects (to keep scores high)
- Feel anxious during sessions
- See learning as compliance rather than curiosity
- Eventually disengage

Nuri's approach: the child feels like they are learning with a fun AI tutor. The parent feels informed and empowered. Neither experience undermines the other.

## Notification Generation Pipeline

### Daily Note Generation (Runs at parent's configured time)

```
1. Query: all parent_child_links
2. For each child:
   a. Aggregate today's session data (quiz_sessions, homework_sessions, voice_logs)
   b. Calculate accuracy per subject
   c. Pull error patterns from Feature 21
   d. Pull confidence data from Feature 19
   e. Estimate session mood from behavioral signals
   f. Check goal progress
   g. Generate AI note using Claude:
      - Input: structured session data + child profile + previous notes
      - Output: personalized daily note in parent's preferred language
   h. Store in daily_summaries
   i. Deliver via parent's chosen channel(s)
3. If child didn't study: send brief "no activity" note (configurable — parent can opt out of these)
```

### Weekly Report Generation (Runs every Sunday at 10:00 AM)

```
1. Query: all parent_child_links
2. For each child:
   a. Aggregate 7 daily_summaries
   b. Calculate trends (compare to previous weekly_report)
   c. Determine topics mastered vs needing work
   d. Pull monthly trend if 4+ weeks of data
   e. Generate AI report using Claude:
      - Input: weekly aggregate + trends + child profile + learning style
      - Output: comprehensive weekly report in parent's language
   f. Store in weekly_reports
   g. Deliver via email (always) + parent's chosen channel(s)
```

### Frustration Alert (Real-time during session)

```
1. After each quiz question, check:
   - Wrong streak >= 4 in a row
   - Session shorter than 50% of average AND accuracy < 40%
   - "Guessed" confidence on 5+ consecutive questions
   - Difficulty was dropped AND session ended within 2 minutes
2. If triggered:
   a. Create frustration_alert record
   b. Generate suggestion using Claude
   c. Send immediate push notification to parent
   d. Nuri auto-adjusts difficulty on child's device
```
